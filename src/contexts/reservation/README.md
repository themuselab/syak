# reservation 컨텍스트 — ⛔ 지금은 포트만 (구현 보류)

예약/슬롯(다음날 빈자리) 도메인. **이번 버전(2.0)에서는 구현하지 않는다.**

## 왜 분리만 해두나
- 실시간 빈 슬롯은 정적 read model에 없는 유일한 데이터.
- 수집은 가게당 로그인+예약창 진입이라 무겁고 봇차단이 잦음 → 버셀 서버리스에서 못 돎.
- 따라서 **별도 백엔드(AWS 예정)** 가 매일 00시 배치로 다음날 슬롯을 미리 계산해 저장하고,
  앱은 그 결과만 조회한다. (읽기/쓰기 분리)

## 지금 상태
- `ports/slot-provider.ts` — 인터페이스만 정의.
- 어댑터(구현) 없음. composition-root에서 주입 안 함.
- UI에서 "맞춤 시간 찾기"는 비활성/티저로 두거나, 수요수집(전화번호) 플로우로 우회.

## 나중에 붙일 때
1. AWS 백엔드가 `SlotProvider`를 구현 (예: `HttpSlotProvider` → API Gateway).
2. `app/composition-root.ts`에서 `slotProvider`를 주입.
3. catalog/ui의 "맞춤 시간 찾기"를 활성화.
   → 나머지 코드(도메인·다른 컨텍스트)는 **한 줄도 안 바뀐다.**
